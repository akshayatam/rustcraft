# 🦀 Rust Chapter 2 – Guessing Game (Key Concepts)

## 🎯 Goal
Build a CLI game where:
- Program generates a random number (1–100)
- User keeps guessing
- Program responds: Too small / Too big / You win

---

## 💻 Guessing Game

```rust
use std::io;                   // For user input
use rand::Rng;                 // Trait for random number generation
use std::cmp::Ordering;        // Enum for comparison results

fn main() {
    println!("Guess the number!");

    // Generate random number between 1 and 100
    let secret_number = rand::thread_rng().gen_range(1..=100);

    loop {
        println!("Please input your guess.");

        // Create mutable string to store input
        let mut guess = String::new();

        // Read user input into 'guess'
        io::stdin()
            .read_line(&mut guess)
            .expect("Failed to read line");

        // Convert String → u32
        let guess: u32 = match guess.trim().parse() {
            Ok(num) => num,        // Valid number
            Err(_) => continue,    // Invalid input → skip iteration
        };

        println!("You guessed: {}", guess);

        // Compare guess with secret number
        match guess.cmp(&secret_number) {
            Ordering::Less => println!("Too small!"),
            Ordering::Greater => println!("Too big!"),
            Ordering::Equal => {
                println!("You win!");
                break;              // Exit loop when correct
            }
        }
    }
}
```

## 🧠 Key Concepts

### 1. Variables & Mutability
- `let` → immutable by default
- `let mut` → mutable variable

### 2. String Handling
- `String::new()` → creates empty string
- `read_line(&mut guess)` → appends input
- `trim()` → removes newline

### 3. Type Conversion
```rust
guess.trim().parse()
```
- Converts `String` → number
- Returns `Result`

### 4. Result & Error Handling
`Resul~t~ has two variants:
- `Ok(value)` → success
- `Err(error)` → failure
Two approaches:
- `.expect()` → crash on error
- `match` → handle error safely

### 5. Match Expression
```rust
match value {
    pattern => action,
}
```
- Exhaustive pattern matching
- Core Rust control flow tool

### 6. Enums
`Ordering` enum:
- `Less`
- `Greater`
- `Equal`

### 7. Random Numbers (External Crate)
Add to `Cargo.toml`:
```toml
rand = "0.8"
```
Use:
```rust
rand::thread_rng().gen_range(1..=100)
```

### 8. Looping
```rust
loop {
    // runs forever until break
}
```
- `break` → exit loop
- `continue` → skip iteration

### 9. Shadowing
```rust
let guess = String::new();
let guess: u32 = ...
```
- Reuse same variable name
- Often used for type conversion

### 10. References
```rust
&mut guess
```
Borrow variable instead of copying
`&mut` → mutable reference

## ⚡ Key Takeaways
Rust enforces type safety strictly
Error handling is explicit via Result
`match` is central to Rust logic
Cargo makes dependency management easy
Rust favors safety + clarity over shortcuts
